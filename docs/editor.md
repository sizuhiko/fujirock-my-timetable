# マイタイムテーブル編集

以下の機能を持つものとします。

- フジロック公式タイムテーブルページをスクレイピングして、同じようなタイムテーブルを表示できる
  - 1日目: https://www.fujirockfestival.com/artist/timetable/24
  - 2日目: https://www.fujirockfestival.com/artist/timetable/25
  - 3日目: https://www.fujirockfestival.com/artist/timetable/26
- タイムテーブルは変更される場合があるので、編集画面からリロードボタンで再同期できること
- ステージごとに色分けされていること。色は公式サイトのタイムテーブルの色と同じものを使うこと
- 見たいライブを選択状態にすると、マイタイムテーブルに追加されること
- 選択状態のライブを未選択にすると、マイタイムテーブルから削除されること
- 隙間時間をクリックすると、フードエリアへ行く予定がマイタイムテーブルに追加できること
  - フェスごはん: https://www.fujirockfestival.com/news/feat07
  - 上記サイトをスクレイピングして、同じような表示から行くエリアを選択できること
  - エリアを選択したときは、開始時間と終了時間を選択できること
  - フェスごはんの予定枠の背景色は白色にすること
  - フェスごはんの予定枠にゴミ箱アイコンを付けて、それがクリックされた場合はマイタイムテーブルから削除できること
- フードエリアへ行く予定を追加できるルールとして、隙間時間だけでなく最初のライブ前と最後のライブ後でもマイタイムテーブルに追加できること
- マイタイムテーブル閲覧ページへ遷移できる

## 実装メモ

- GitHub Pages は静的ホスティングのため常駐サーバーを持てない。スクレイピングはビルド時に Node.js スクリプトで実行し、結果を静的 JSON としてリポジトリにコミットする方式とする
- GitHub Actions で定期実行(スケジュール)し、公式サイトの更新を定期的に取り込んで自動デプロイする
- 編集画面の「リロードボタン」は、実行時に公式サイトへ直接アクセスするのではなく、同一オリジンの最新の静的 JSON を再取得する方式とする(CORS の問題を回避し、会場 Wi-Fi でも安定動作させるため)

## UI の改善メモ

ここは編集画面のUIに関する改善メモを記録します。

- 公式タイムテーブルとマイタイムテーブルが縦スクロールをしていて、マイタイムテーブルの存在に気づかない可能性があるので、以下のようにしてください
  - 1日目、2日目、3日目の下あたりに、ページ下へのマイタイムテーブルへスクロールするボタンまたはアンカーリンクを置く
  - ページの右下あたりにTOPまでスクロールして戻るアクションボタン（アイコンのみでok）を置く
- 公式タイムテーブルの下でフードエリアの予定を追加できるが、そのエリアのタイトルやアンカーが `マイタイムテーブル` になっているので、 `フードエリア` としたい
